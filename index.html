Introduction

JavaScript est un langage de script multiplateforme orienté objet. C'est un langage petit et léger. Dans un environnement hôte (par exemple, un navigateur Web), JavaScript peut être connecté aux objets de son environnement pour fournir un contrôle programmatique sur eux.

JavaScript contient une bibliothèque standard d'objets, tels que Array, Date et Math, et un ensemble de base d'éléments de langage tels que des opérateurs, des structures de contrôle et des instructions. Core JavaScript peut être étendu à diverses fins en le complétant avec des objets supplémentaires ; par exemple:

  • JavaScript côté client étend le langage de base en fournissant des objets pour contrôler un navigateur et son modèle d'objet de document (DOM). Par exemple, les extensions côté client permettent à une application de placer des éléments sur un formulaire HTML et de répondre aux événements de l'utilisateur tels que les clics de souris, la saisie de formulaire et la navigation dans les pages.
  • JavaScript côté serveur étend le langage de base en fournissant des objets pertinents pour l'exécution de JavaScript sur un serveur. Par exemple, les extensions côté serveur permettent à une application de communiquer avec une base de données, d'assurer la continuité des informations d'un appel à l'autre de l'application ou d'effectuer des manipulations de fichiers sur un serveur.
Prérequis

Ce guide suppose que vous disposez des connaissances de base suivantes :

  • Une compréhension générale d'Internet et du World Wide Web (WWW).
  • Bonne connaissance pratique du langage de balisage hypertexte (HTML).
  • Une certaine expérience en programmation. Si vous débutez dans la programmation, essayez l'un des tutoriels liés sur la page principale à propos de JavaScript.
JavaScript et Java

JavaScript et Java sont similaires à certains égards, mais fondamentalement différents à d'autres. Le langage JavaScript ressemble à Java mais n'a pas le typage statique et la vérification de type forte de Java. JavaScript suit la plupart des syntaxes d'expression Java, des conventions de dénomination et des constructions de flux de contrôle de base, raison pour laquelle il a été renommé de LiveScript en JavaScript.

Contrairement au système de classes de compilation de Java construit par des déclarations, JavaScript prend en charge un système d'exécution basé sur un petit nombre de types de données représentant des valeurs numériques, booléennes et de chaîne. JavaScript a un modèle d'objet basé sur un prototype au lieu du modèle d'objet basé sur une classe plus courant. Le modèle basé sur un prototype fournit un héritage dynamique ; c'est-à-dire que ce qui est hérité peut varier d'un objet à l'autre. JavaScript prend également en charge les fonctions sans aucune exigence déclarative particulière. Les fonctions peuvent être des propriétés d'objets, s'exécutant comme des méthodes faiblement typées.

JavaScript is a very free-form language compared to Java. You do not have to declare all variables, classes, and methods. You do not have to be concerned with whether methods are public, private, or protected, and you do not have to implement interfaces. Variables, parameters, and function return types are not explicitly typed.

Variables

Vous utilisez des variables comme noms symboliques pour les valeurs de votre application. Les noms des variables, appelés identificateurs, sont conformes à certaines règles.

Un identificateur JavaScript doit commencer par une lettre, un trait de soulignement (_) ou signe dollar ($); les caractères suivants peuvent également être des chiffres (0-9). Comme JavaScript est sensible à la casse, les lettres incluent les caractères « A » à « Z » (majuscules) et les caractères « a » à « z » (minuscules).

Vous pouvez utiliser iso 8859-1 ou des lettres Unicode telles que å et ü dans Identificateurs. Vous pouvez également utiliser les séquences d’échappement Unicode comme caractères dans les identificateurs. Voici quelques exemples de noms légaux : Number_hits, temp99 et _name.

Déclaration variable
Vous pouvez déclarer une variable de trois manières :

Avec le mot-clé var. Par exemple, var x = 42. Ceci la syntaxe peut être utilisée pour déclarer des variables locales et globales.

En lui attribuant simplement une valeur. Par exemple x = 42.Cela déclare toujours une variable globale. Il génère un avertissement JavaScript strict. Vous ne devriez pas l’utiliser variante.

Avec le mot-clé let. Par exemple, soit y = 13. Ceci la syntaxe peut être utilisée pour déclarer une variable locale d’étendue de bloc. Voir "Portée variable" ci-dessous.

Portée variable

Lorsque vous déclarez une variable en dehors d’une fonction, elle est appelée une global variable, car elle est disponible pour tout autre code dans le document en cours. Lorsque vous déclarez une variable dans une fonction, elle est appelée variable locale, car elle n’est disponible qu’à l’intérieur de cette fonction.

JavaScript avant ECMAScript 2015 n’a pas d’instruction block champ d’application; au contraire, une variable déclarée dans un bloc est locale à l' (ou étendue globale) dans laquelle réside le bloc. Pour Par exemple, le code suivant sera enregistré 5, car l’étendue de x est le (ou contexte global) dans lequel x est déclaré, et non le block, qui dans ce cas est une instruction if.

if (true) { var x = 5; } console.log(x); // 5

This behavior changes, when using the let declaration introduced in ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Variables globales

Les variables globales sont en fait des propriétés de l’objet global. Dans le web pages l’objet global est fenêtre, afin que vous puissiez définir et accéder à global à l’aide de la syntaxe window.variable.

Par conséquent, vous pouvez accéder aux variables globales déclarées dans une fenêtre ou cadrer à partir d’une autre fenêtre ou d’un autre cadre en spécifiant la fenêtre ou nom du cadre. Par exemple, si une variable appelée phoneNumber est déclarée dans un document, vous pouvez faire référence à cette variable à partir d’un iframe en tant que parent.phoneNumber.

Constances

Vous pouvez créer une constante nommée en lecture seule avec le mot-clé const. La syntaxe d’un identificateur constant est la même que pour une variable identificateur : il doit commencer par une lettre, un trait de soulignement ou un signe dollar et peut contenir des caractères alphabétiques, numériques ou de soulignement.

const PI = 3.14;

Une constante ne peut pas changer de valeur par affectation ou être déclarée à nouveau pendant l’exécution du script. Il doit être initialisé à une valeur.

Les règles d’étendue pour les constantes sont les mêmes que celles pour le bloc let variables d’étendue. Si le mot-clé const est omis, l’identificateur est supposé représenter une variable.

Vous ne pouvez pas déclarer une constante portant le même nom qu’une fonction ou dans la même portée. Par exemple:

Type de données

La dernière norme ECMAScript définit sept types de données :

  • Six types de données qui sont des primitives :

    • Booléen. vrai et faux
    • zéro. Mot-clé spécial indiquant une valeur null. Parce que JavaScript est sensible à la casse, null n’est pas la même chose que Null, NULL ou toute autre variante.
    • indéfini. Propriété de niveau supérieur dont la valeur n’est pas définie.
    • Nombre. 42 ou 3.14159.
    • String. "Howdy"
    • Symbole (nouveau dans ECMAScript 2015). Un type de données dont les instances sont uniques et immuables.
  • et Object
Bien que ces types de données soient relativement faibles, ils permettent vous pour effectuer des fonctions utiles avec vos applications. Objets et sont les autres éléments fondamentaux du langage. Déclaration if else
Utilisez l’instruction if pour exécuter une instruction si une condition logique est vrai. Utilisez la clause else facultative pour exécuter une instruction si le est false. Une instruction if se présente comme suit : if (condition) { statement_1; } else { statement_2; } La condition peut être n’importe quelle expression qui s’évalue à true ou false. Voir Booléen pour une explication de ce qui est évalué à vrai et faux. Si condition s’évalue à true, statement_1 est exécuté ; autrement statement_2 est exécutée. statement_1 et statement_2 peuvent être n’importe quel , y compris d’autres instructions if imbriquées.

Vous pouvez également composer les instructions en utilisant else si vous souhaitez en avoir plusieurs conditions testées dans l’ordre, comme suit:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } Dans le cas de conditions multiples, seule la première condition logique qui évalue à true sera exécuté. Pour exécuter plusieurs , regroupez-les dans une instruction block ({ ... }) . Dans En général, il est recommandé de toujours utiliser des instructions de bloc, en particulier lors de l’imbrication des instructions if : if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } Il est conseillé de ne pas utiliser de simples affectations dans un conditionnel expression, car l’affectation peut être confondue avec l’égalité lorsque en jetant un coup d’œil sur le code. Par exemple, n’utilisez pas le code suivant : if (x = y) { /* statements here */ } Si vous devez utiliser une affectation dans une expression conditionnelle, une pratique courante consiste à mettre parenthèses supplémentaires autour de l’affectation. Par exemple: if ((x = y)) { /* statements here */ }
Déclaration while
Une instruction while exécute ses instructions tant qu’une instruction spécifiée condition est évalué à true. Une instruction while se présente comme suit : while (condition) statement Si la condition devient false, l’instruction dans la boucle cesse de s’exécuter et le contrôle passe à l’instruction qui suit la boucle.

Le test de condition se produit avant l’exécution de l’instruction dans la boucle. Si la condition renvoie true, l’instruction est exécutée et le L’état est testé à nouveau. Si la condition renvoie false, exécution arrête et le contrôle est transmis à l’instruction while suivante.

Pour exécuter plusieurs instructions, utilisez une instruction block ({ ... }) pour regroupez ces déclarations.

Exemple:

The following while loop iterates as long as n is less than three:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

À chaque itération, la boucle incrémente n et ajoute cette valeur à x. Par conséquent, x et n prennent les valeurs suivantes :

  • After the first pass: n = 1 and x = 1
  • After the second pass: n = 2 and x = 3
  • After the third pass: n = 3 and x = 6

Après avoir terminé le troisième passage, la condition n < 3 n’est plus true, de sorte que la boucle se termine.

Déclaration de fonctions
Une définition de fonction (également appelée déclaration de fonction, ou fonction statement) se compose du mot-clé function, suivi de :
  • Nom de la fonction.
  • Liste des arguments de la fonction, entre parenthèses et séparés par des virgules.
  • Les instructions JavaScript qui définissent la fonction, incluses dans crochets bouclés, { }.

Par exemple, le code suivant définit une fonction simple nommée carré:

function square(number) { return number * number; }

The function square takes one argument, called number. The function consists of one statement that says to return the argument of the function (that is, number) multiplied by itself. The return statement specifies the value returned by the function.

return number * number;

Les paramètres primitifs (tels qu’un nombre) sont transmis aux fonctions par valeur; la valeur est transmise à la fonction, mais si la fonction modifie la valeur du paramètre, cette modification n’est pas reflétée globalement ou dans la fonction d’appel.
